# 階段交付物規格定義

> **目的**: 明確定義各階段目錄應放置的交付物類型、格式標準與品質要求

---

## 📁 01_raw/ - 原始文件收集區

### 目的與定位
- **階段定義**: 知識治理流程的起點，收集所有「可能有價值」的原始文件
- **治理原則**: 「先收集，後篩選」- 避免遺漏有價值內容
- **品質要求**: 無需加工，保持原始狀態

### 交付物規格

#### 📋 必需交付物
```yaml
目錄結構:
  sample_docs/          # 教學範例文檔
  project_reports/      # 專案報告收集區
  meeting_notes/        # 會議記錄收集區
  knowledge_candidates/ # 待評估文件
  external_sources/     # 外部來源文件

檔案命名規範:
  格式: "{日期}_{專案/會議名稱}_{文件類型}_{版本}.{副檔名}"
  範例: "2025-11-17_Server散熱設計_技術規格_v1.2.pdf"
```

#### 📊 元資料記錄
**收集登記表** (`collection_log.csv`):
```csv
檔案名稱,收集日期,來源部門,提交人,文件類型,初步評估,備註
2025-11-17_Server散熱設計_技術規格_v1.2.pdf,2025-11-17,RD部,張工程師,技術文件,高價值,客戶專案產出
```

#### ✅ 品質標準
- **完整性**: 檔案可正常開啟，內容完整
- **可讀性**: 文字清晰，圖表可識別
- **來源性**: 明確標記文件來源與提交人
- **時效性**: 收集後72小時內完成初步分類

## 📄 02_processed/ - 一頁式技術卡

### 目的與定位
- **階段定義**: 將原始文件精煉成「可複用的知識資產」
- **治理原則**: 「精而不雜」- 每張技術卡解決一個明確問題
- **品質要求**: 符合四要素框架，通過同儕審核

### 交付物規格

#### 📝 技術卡標準格式
**檔案命名**: `{YYYY-MM}_{問題領域}_{解決方案簡述}_{適用範圍}.md`

**內容結構**:
```markdown
# 技術卡：{標題}

## 📋 基本資訊
- **技術卡ID**: TC-{YYYY-MM-DD}-{序號}
- **建立日期**: {日期}
- **建立者**: {姓名}
- **來源文件**: {原始文件路徑}
- **分類標籤**: #{Domain} #{Type} #{Confidentiality}

## ❓ 問題描述
{清楚描述要解決的問題，包含背景與痛點}

## 💡 解決方案
{具體的解決方法、步驟或規則}

## 📊 關鍵參數
{重要數據、圖表、公式或配置參數}

## 🎯 適用範圍與限制
- **適用條件**: {什麼情況下可以使用}
- **限制條件**: {什麼情況下不適用}
- **預期效果**: {使用後的預期改善}

## 🔗 相關資源
- **參考文件**: {相關技術卡或文檔連結}
- **聯絡人**: {技術負責人}
- **更新記錄**: {版本變更歷史}
```

#### 📊 品質控制清單
**技術卡檢核表** (`tech_card_checklist.md`):
- [ ] 問題描述清晰具體
- [ ] 解決方案可操作執行
- [ ] 關鍵參數完整準確
- [ ] 適用範圍明確界定
- [ ] 同儕審核已完成
- [ ] 分類標籤正確標記

## 🔧 共用規範與標準

### 檔案命名統一規範
```yaml
技術卡檔案: "{YYYY-MM}_{問題域}_{解決方案}_{適用範圍}.md"
元資料檔案: "{功能}_{版本}.{json|csv|md}"
報告檔案: "{報告類型}_{週期}_{日期}.{md|pdf}"
```

### 目錄結構標準
```
每個階段目錄必需包含:
├── README.md           # 目錄用途與使用說明
├── templates/          # 範本檔案
├── samples/            # 範例資料
└── schemas/            # 資料結構定義(如適用)
```

### 品質管控標準
```yaml
檔案完整性:
  - 所有必填欄位完整
  - 檔案格式符合規範
  - 檔案可正常存取

內容品質:
  - 語言清晰易懂
  - 資訊準確可靠
  - 結構邏輯合理

更新維護:
  - 定期檢查有效性
  - 及時更新過期內容
  - 記錄變更歷史
```
